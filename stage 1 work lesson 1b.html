Lesson 1: BASICS

How The Web Works

The web is a bunch of computers that communicate with each other. When a person goes to a web page. The computer sends a HTTP Request to a server.

The server finds the appropriate HTML document and sends it back to the user's computer where a web browser interprets the page and displays it on the user's screen.

Tags-tells the computer what to do in a way it can understand.

Http - is the communications protocal used mainly on the web.

Servers - computers that host files that make up the web.

Internet - world's largest computer network.

Browser - A program that runs on your computer to interpret and display files found on the web host servers.

Thinking Like A Programmer
  1. Procedural thinking- take big problems and brake them down into smaller steps.
  2. Abstract thinking- apple + apple = 2 apples, orange + orange = 2 oranges abstract concept (1+1=2)
  3. Systematic Debugging- Identify problems and track down causes and fixes.
  4. Technologic Empathy- uderstand how a computer sees things.
  5. Systems thinking- take one big idea and brake it dowon into smaller steps.
HTML

HTML stands for Hypertext Markup Language. HTML documents form the majority of the content on the web. HTML documents contain ""text content" which describes "what you see" and markup which describes "how it looks".

Inline / Block Elements

HTML elements are either inline or block. Block elements form an "invisible box" around the content inside of them.

Lesson 2: Web Pages, Documents, And Structure

Creating Structured Documents With HTML
Important Things To Take Away
  1. HTML & CSS are both "Languages"
  2. HTML controls the "structure" of a web page
  3. CSS controls the "style" of a web page (How it looks)
  4. when programmers talk about the "DOM" they are talking about the tree-like structure of a page
DOM - Document Object Model

All elements on a web page are boxes. The "tree-like structure" comes from the fact that HTML elements can have other elements inside of them. browsers display this structure as a series of nested boxes.

How To Listen To Experts
Start by coding big boxes first then add the smaller subboxes. The Title and a graphic might be the first box. Then the body containing text and url links with graphics or video or music would be the second big box. Then break the graphics and text and other elements into smaller boxes to code.
The Tools Programmers Use To Write Code

-scratchpad for small projects

-sublime text 2 for larger more complex projects

HTML Structure

Like branches of a tree, HTML elements can have element boxes nested inside them.

The Relationship Between Indented HTML And Boxes

The more indented a element is; the more deeply nested it's corresponding "box" will be.

Lesson 3:Box Sizing

Four Main Ideas

  1. HTML elements are boxes and each box has 4 components- margins, borders, padding, and content.
  2. Because there are so many components to each box, it can often be hard to get the size of the box right.
  3. There are two techniques you can use to help deal with sizing issues. set sizes interms of percent rather than pixels.
    set the box-sizing attribute to border-box for every element.
  4. Different browsers work slightly differently. Sometimes this causes different browsers to display the same code differently.


ex. Style.css <*{ box-sizing: border-box;>

Box Positioning
  1. Divs are block elements (as opposed to inline elements). By default they take up the entire width of a page.
  2. adding the rule [display-flex;] to the appropriate css will over ride this behavior and let divs appear next t eachother.
Code,Test,Refine

  1. look for natural boxes
  2. look for repeated styles and semantic elements
  3. write your HTML
  4. apply styles (from biggest to smallest)
  5. fix the small things
Avoiding Repetition

Avoiding Errors: Avoiding repetition helps redue the likely hood of not consistantly entering code over and over and repeating certain style attributes into many HTML elements or code. Errors can be eaily made when programmers make changes to the code.

Consistency: By assigning each lesson to look the same. I was able to assign each lesson div the same class and specify the style for that class in my css file which makes changes much easier and avoid making duplication errors.

CSS

CSS stands for Cascading Style Sheets and they give programmers a way to control the style of related HTML elements. This is done by giving similar HTML elements the same class name and then specifying the style that should apply to that class.
